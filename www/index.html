JFlap Bots in Maize

Additional UI Elements in Maize

When Maize is loaded with JFlap compatible bots, two additional windows will be opened alongside the main Maize UI, one for Push Down Automata (PDABot) and the other for Finite State Machines (FSMBot).
Each window will have it's title set to the name of the bot it controls - a randomly generated [adjective] [colour] [animal] combination, these can be matched with the bot instance in Maize's UI to run the bot in various mazes.

Maize's bot list, with a PDABot selected

The random names are used to identify which bot instance in Maize matches to which window, and are mainly useful for pitting two bots of the same type against each other with different JFlap graphs (PDABot vs PDABot, for example).

Running Graphs in Maize

When started, each bot in the current application instance will open a new window.
The two buttons at the top of each of these windows allow you to load or reload graphs designed in JFlap.

A PDABot window, annotated

To get the bots running, fire up JFlap and lay out your bot logic, then launch Maize and find the correct bot type for your graph.

  • Push Down Automata should be loaded into a PDABot
  • Finite State Machines should be loaded into a FSMBot

If you see a message complaining that the bot doesn't understand the graph type you have chosen, make sure you're using the right bot with the right graph.

The "Run Tests" tab, annotated

Next, choose the "Run Tests" tab in Maize, find the bot you're using in the left hand list and pick a maze for it to solve from the right-hand list

Select "New Test" to load a new test into Maize with the bot(s) and maze you selected, and press "(Re)Start" to start the test.

Log messages from a Push Down Automata running in PDABot

Messages from PDABot or FSMBot will be shown in the output log in their windows, and show what the bot evaluated during it's turn, including:

  • The node the bot started at and any subsequent nodes it went through this turn
  • The rules the bot evaluated at each node
  • Which rule the bot actually took (in the case of nondeterministic options)
  • What actions the bot has performed (Forward, Back, Left or Right, if any)

The window only shows the last update from the bots, but messages are also written to the console/terminal, so if you're having trouble getting a bot to do what you want, it might be worth running Maize from the command line and going through the log messages
